# Code Clone Detection Pipeline

This directory contains the experimental framework to evaluate and compare embeddings generated by transformer-based models for code clone detection.

## Key Capabilities
1. **Fine-tuning:** Optimize **GraphCodeBERT** and **CodeT5+** models using Siamese network architectures.
2. **Embedding Generation:** Convert source code into high-dimensional vectors using:
   - GraphCodeBERT (Base & Tuned)
   - CodeT5+ (Base & Tuned)
   - Llama 3.2 1B
3. **Classification:** Train an **XGBoost** classifier on the generated embeddings.
4. **Visualization:** Generate performance metrics and plots, including ROC curves and Confusion Matrices.

---

## Prerequisites

To successfully use **GraphCodeBERT**, you must clone the official CodeBERT repository into this directory:
```bash
git clone https://github.com/microsoft/CodeBERT.git
```

## Usage Workflow

The pipeline must be executed in a specific linear order. If you wish to test a fine-tuned version of a model, you must complete the training phase before generating embeddings.

## Correct Execution Order

### 1. Fine-Tuning 
If you want to test a fine-tuned model, you must run the tuner first.  

**Example:**  
```bash
python codet5p_fine_tuner.py
# or
python graphcodebert_fine_tuner.py
```

### 2. Embedding Generation 
Generate the vector representations for your training and validation sets using a base model or the weights from the previous step.
**Example:** 
```bash
python codet5p_embedder.py
# or
python graphcodebert_fine_tuner.py
# or
python llama_fine_tuner.py
```
### 3. Classification & Evaluation
Train the XGBoost head on the generated .jsonl embeddings and produce final performance figures.
```bash
python XGBoost.py
```
